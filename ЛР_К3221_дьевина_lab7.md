## Цель работы

Разработать веб-приложение, которое принимает данные пользователя и строку подключения к MongoDB, после чего записывает документ в коллекцию `users`.

## Что реализовано

- `GET /login/` возвращает логин Moodle: `dieuvina`.
- `POST /insert/` принимает `application/x-www-form-urlencoded` поля:
  - `login`;
  - `password`;
  - `URL`.
- Приложение подключается к MongoDB по переданному `URL`.
- В коллекцию `users` записывается документ `{ login, password }`.
- В ответ возвращается JSON с признаком успешной вставки и `insertedId`.
- Добавлены CORS-заголовки для `https://kodaktor.ru`.
- Добавлен `Dockerfile` для деплоя на Render.

## Как проверить

После деплоя заменить `<URL>` на адрес Render.

Проверить логин:

```bash
curl <URL>/login/
```

Проверить вставку в MongoDB:

```bash
curl -X POST <URL>/insert/ \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "login=test_user&password=test_password&URL=mongodb://user:password@host/database"
```

Ожидаемый результат:

```json
{"inserted":true,"insertedId":"..."}
```

Если строка подключения неверная или база недоступна, приложение вернет JSON с текстом ошибки.
